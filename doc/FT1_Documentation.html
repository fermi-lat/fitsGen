FT1 DOCUMENTATION

General Comments

The LEVEL1 database contains all the event information necessary for scientific analysis of the LAT data.
The scientific analysis will use FITS representation of  the LEVEL1 database, and its template format is named FT1.

Currently, meritAlg.cxx in the merit package computes all the necessary fields and loads them into the merit.root file.
The fitsGen package has 2 executables, named makeFT1.exe and makeFT2.exe, which will parse the former ROOT file and
build the FT1 and FT2 FITS files.

The names of the fields in the following list follows the FT1 definition list, see the FT1 Definition Page for more details.
Red fields means not yet included in the FT1 fits file.

Description of FT1 fields

There are 2 extensions in an FT1 fits file, the first one is the main one, and takes all the remaining of this page; the second extension has the following entries:

 Unit

 format
EVENT_ID NONE
 0.0 - 2147483647 4-byte signed INTEGER
Description:
    The offline reconstruction software use the pair (RUNID, EVENTID) to define an event. The  FT1 field EVENT_ID 'concatenates' these 2 values into a single one:
        EVENT_ID = RUNID*N + EVENTID
N represents the maximum number of events that can be saved in one run. It is currently defaulted to 100000 (see meritAlg.cxx for definition of m_nbOfEvtsInFile)

CALIB_VERSION
 NONE
 N/A
 triplet of 2-byte INTEGER
Description:
    This entry is a vector of 3 elements. For DC1 it has been used to record the selection flags:
  •         CALIB_VERSION[0] is 1 if the event passed the background cuts, 0 otherwise;
  •         CALIB_VERSION[1] is 1 if the event passed the PSF  filter cuts, 0 otherwise;
  •         CALIB_VERSION[2] is 1 if the event passed the CAL filter cuts, 0 otherwise;
See irfAnalysis package for a definition of these cuts.

Reconstruction Data:

   These are the fields directly depending on the reconstructed information. It can happen that information from reconstruction is inadequate, for instance when the event is a  'CAL  ONLY' trigger. In  such cases some fields are ill defined and defaulted to 'GUARD' values.
    Currently, the reconstruction information is taken from either the best reconstructed vertex (i.e. gamma) or the best reconstructed track, depending on the value of the following cut:
VtxAngle == 0.0  ||  IMvertexProb < 0.5 
    This cut is defined in the ClassificationTree class, in the meritAlg package.

Description:
    Take the direction from the center of the earth to the center of the satellite: this defines the earth zenith (up to an oblateness correction which is not included). ZENITH_ANGLE
